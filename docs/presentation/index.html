What Are AI Agents?

  • Autonomous decision-making — choose actions based on context, not just follow scripts
  • Tool use — search databases, call APIs, scrape websites on demand
  • Stateful reasoning — maintain memory and context across multiple steps
  • Beyond simple LLM calls — orchestrate multi-step workflows with branching logic
An agent decides what to do next based on what it learns along the way.

Why LangGraph?

  • Graph-based orchestration — define agent logic as nodes and edges, not linear chains
  • Conditional routing — branch execution based on LLM decisions at runtime
  • Built-in state management — typed state flows automatically through every node
  • Observability out of the box — LangSmith tracing + LangGraph Studio for visual debugging
Plain chains are linear: A → B → C. Agents need graphs — with branches and decisions.

LangGraph Core Concepts

◉ Nodes

Each node is a Python function that receives the current state, does work (call an LLM, query a database), and returns updates to the state.

⟶ Edges

Normal edges always go A → B.
Conditional edges pick the next node based on a function — this is how the agent makes decisions.

📋 State

A Python TypedDict that all nodes share. Each node reads what it needs and writes back its results. Think of it as the agent's working memory.

💬 Messages

HumanMessage (user input) and AIMessage (agent response). The add_messages reducer keeps the full conversation history.

# How it all wires together
graph = StateGraph(AgentState, input=InputState)
graph.add_node("classify", classify_query_node)    # add a node
graph.add_node("generate", generate_answer_node)
graph.add_conditional_edges("classify", route)     # conditional routing
app = graph.compile()                              # ready to run!
